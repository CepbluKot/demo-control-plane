# Work done on 2026-06-29

Repository: `llm-summary-generator`

## Pipeline and LLM execution

- Added a shared LLM dispatch/pool concept so multiple jobs do not independently flood the LLM gateway.
- Added per-job LLM concurrency metadata support:
  - frontend sends `metadata.llm_concurrency`;
  - backend accepts `llm_concurrency`, `llm_max_concurrency`, and `max_concurrent_llm_calls`;
  - backend clamps requested value to env `llm_max_concurrency`.
- Set default LLM timeout to 10 minutes.
- Added safer worker behavior when the LLM pool is busy.
- Added job rerun and individual node rerun support.

## Structured final output

- Added support for user-provided final output schema through `metadata.output_json_schema`.
- Backend uses that schema for FINAL structured LLM response when possible.
- If the provided object is already JSON Schema, backend normalizes it.
- If the object is a shape/example, backend converts it to JSON Schema.
- Final prompt receives the schema text through `{output_json_schema}`.
- Report format instruction metadata is merged into FINAL prompt context.

## Telemetry, progress, and recovery

- Enhanced progress snapshots with node/job telemetry useful for frontend graph and result display.
- Added timing and token metadata support for pipeline/node reporting.
- Improved recovery behavior for service/worker interruption.
- Added safer handling of failed nodes and final errors.
