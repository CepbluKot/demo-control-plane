0 Создать Workflow

  • Dify → Studio → Create App → выбрать Workflow
  • Название: Log Incident Analyzer
  • Откроется пустой граф с нодой Start и End

1 Переменные среды

Dify → Settings → Environment Variables → добавить:

CH_HOST=clickhouse.internal
CH_PORT=8123
CH_USER=default
CH_PASSWORD=ваш_пароль
CH_DATABASE=default
LLM_API_BASE=http://vllm.internal:8000
LLM_API_KEY=none
LLM_MODEL=llama-3-70b
LLM_TIMEOUT=2400
GROUP_SIZE=4
BATCH_SIZE=200
CHUNK_TOKEN_BUDGET=6000
⚠️ LLM_API_BASE — без /v1 на конце, нода добавит его сама

2 Start Node

START NODE

Нажать на ноду Start → секция Input Fields → добавить 4 поля:

Имя поляТип
period_startString — начало периода, ISO 8601 (напр. 2024-01-15T10:00:00)
period_endString
incident_infoString — описание инцидента
alertsString — тексты алертов (можно пустая строка)
✓ КАК ПРОВЕРИТЬ

Start node нельзя запустить отдельно — проверяется в шаге 3 вместе с Load&Chunk

3 Code Node: Load & Chunk

CODE NODE

Создать ноду: нажать + после Start → выбрать Code

1. Вставить код — скопировать весь файл dify/nodes/load_chunk.py в редактор

2. Input Variables (вкладка Inputs слева):

period_start← start.period_start
period_end← start.period_end
incident_info← start.incident_info
alerts← start.alerts

3. Output Variables:

batchesArray[String]
batch_countNumber
✓ КАК ПРОВЕРИТЬ

Нажать ▶ Run на ноде → ввести в появившейся форме:

{
  "period_start": "2024-01-15T10:00:00",
  "period_end": "2024-01-15T12:00:00",
  "incident_info": "pods falling in namespace airflow",
  "alerts": "AirflowKubernetesExecutorFailed fired at 10:15"
}

Ожидаем:

{
  "batches": ["[2024-01-15T10:00:01 → ...] ×3  airflow/pod-xxx  ERROR ...", ...],
  "batch_count": 12
}
✅ Если batch_count > 0 — ClickHouse отвечает и данные загружены
⚠️ Если batches = [] — проверь что период реальный и данные в CH есть
⚠️ Если ошибка подключения — проверь CH_HOST/CH_PORT в env vars

4 Iteration Node: MAP

ITERATION NODE (parallel)

Создать ноду: нажать + → выбрать Iteration

Input Array{{load_chunk.batches}}
Parallel mode✅ включить
Max concurrency4–8 (зависит от LLM)

Внутри Iteration добавить LLM Node:

LLM NODE (внутри Iteration)
Modelвыбрать из настроек Dify
Output formatJSON — включить JSON mode

SYSTEM prompt (вставить как есть, переменные Dify заменят значения):

You are a senior SRE analyzing a production log batch during an active incident.

=== Incident context ===
{{start.incident_info}}

Period under investigation: {{start.period_start}} → {{start.period_end}}

=== Active alerts ===
{{start.alerts}}

=== Language ===
Think and respond in English. Keep technical terms as-is (OOM, pod names,
Kubernetes objects, error codes). The incident context may be in Russian.

=== Task ===
Analyze the log batch. Extract events, evidence, hypotheses.
Output ONLY a JSON object, no prose.

=== Output schema ===
{
  "time_range": ["<ISO8601>", "<ISO8601>"],
  "narrative": "<2-4 sentences: what happened>",
  "events": [
    {
      "id": "<evt-NNN-SEQ>",
      "timestamp": "<ISO8601>",
      "source": "<pod/service>",
      "severity": "<critical|high|medium|low|info>",
      "message": "<что случилось>",
      "raw_refs": []
    }
  ],
  "hypotheses": [
    {
      "id": "<hyp-NNN-SEQ>",
      "statement": "<гипотеза>",
      "confidence": "<high|medium|low>",
      "supporting_event_ids": []
    }
  ],
  "evidence": [{"id": "<ev-NNN-SEQ>", "text": "<факт>", "event_ids": []}],
  "gaps": ["<что не видно в этом батче>"],
  "alert_refs": []
}

USER prompt:

=== Log batch ===
{{item}}

{{item}} — текущий батч из Iteration, подставляется автоматически

Output Variable Iteration: Dify автоматически создаёт переменную с результатами — она будет называться map.output или iteration_1.output (зависит от версии)

✓ КАК ПРОВЕРИТЬ

Вариант А — тест одной итерации: Нажать ▶ Run на LLM Node внутри, ввести один батч вручную:

item: "[2024-01-15T10:05:01 → 10:05:03] ×5  airflow/scheduler-xxx  ERROR Failed to execute task"
start.incident_info: "airflow scheduler crash"
start.period_start: "2024-01-15T10:00:00"
start.period_end: "2024-01-15T12:00:00"
start.alerts: "AirflowSchedulerDown"

Ожидаем: валидный JSON с полями time_range, narrative, events

✅ Если вернулся JSON — MAP нода работает
⚠️ Если вернулся текст не-JSON — включи JSON mode в настройках LLM Node

Вариант Б — запустить всё до MAP: нажать Run на Iteration ноде, она возьмёт batches из предыдущей ноды

5 Code Node: REDUCE

CODE NODE

Создать ноду: нажать + после Iteration → выбрать Code

1. Вставить код — скопировать весь файл dify/nodes/reduce.py

2. Input Variables:

map_results← вывод Iteration ноды (Array of Objects)
incident_info← start.incident_info
period_start← start.period_start
period_end← start.period_end

3. Output Variables:

merged_analysisObject
⚠️ Важно: map_results должен быть Array of Objects, не Array of Strings. Убедись что LLM Node в MAP отдаёт JSON-объект, а не строку
✓ КАК ПРОВЕРИТЬ

Нажать ▶ Run → ввести мокнутые map_results:

{
  "map_results": [
    {
      "time_range": ["2024-01-15T10:00:00", "2024-01-15T10:30:00"],
      "narrative": "Scheduler crashed due to OOM",
      "events": [{"id": "evt-001-001", "timestamp": "2024-01-15T10:05:00",
                  "source": "airflow-scheduler", "severity": "critical",
                  "message": "OOMKilled", "raw_refs": []}],
      "hypotheses": [], "evidence": [], "gaps": [], "alert_refs": []
    },
    {
      "time_range": ["2024-01-15T10:30:00", "2024-01-15T11:00:00"],
      "narrative": "Workers failing to pull image",
      "events": [{"id": "evt-002-001", "timestamp": "2024-01-15T10:35:00",
                  "source": "worker-pod", "severity": "high",
                  "message": "ImagePullBackOff", "raw_refs": []}],
      "hypotheses": [], "evidence": [], "gaps": [], "alert_refs": []
    }
  ],
  "incident_info": "airflow crash",
  "period_start": "2024-01-15T10:00:00",
  "period_end": "2024-01-15T12:00:00"
}

Ожидаем:

{
  "merged_analysis": {
    "narrative": "...",
    "events": [{...}, {...}],
    ...
  }
}
✅ Если merged_analysis содержит события из обоих батчей — REDUCE работает
⚠️ Если timeout — проверь LLM_API_BASE и LLM_MODEL в env vars

6 End Node

END NODE

Нажать на End Node → добавить output:

merged_analysis← reduce.merged_analysis

7 Полный запуск

Нажать ▶ Run на всём Workflow → ввести параметры:

{
  "period_start": "2024-01-15T10:00:00",
  "period_end": "2024-01-15T12:00:00",
  "incident_info": "Airflow scheduler falling, ImagePullBackOff on worker pods in namespace airflow",
  "alerts": "AirflowKubernetesExecutorFailed fired 10:15\nImagePullBackOff on ndp-p01"
}
✅ Успех: End node возвращает merged_analysis с narrative, events и hypotheses
Время: зависит от количества батчей и LLM. Ожидай 5–30 минут на большой период.
⚠️ Если batch_count = 0 — данных нет за период, попробуй другой период
⚠️ Если REDUCE зависает — проверь LLM_TIMEOUT, увеличь до 3600
⚠️ Если MAP возвращает строки вместо объектов — включи JSON mode в LLM Node
